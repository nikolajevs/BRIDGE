# Игра как Telegram Mini App

Игра открывается прямо внутри Telegram: партия собирается в чате за минуту,
вход — без пароля, статистика привязывается к профилю Telegram.

Тот же сайт продолжает работать в обычном браузере — код один, ничего
дублировать не нужно.

## 1. Создать бота

В Telegram напиши [@BotFather](https://t.me/BotFather):

```
/newbot
```

Он спросит имя (любое, например `Двинский Бридж`) и юзернейм (должен
заканчиваться на `bot`, например `dvinsky_bridge_bot`). В ответ пришлёт
**токен** вида `123456789:AAE...` — это пароль от бота, никому не показывай.

## 2. Привязать игру к боту

Там же, в @BotFather:

```
/newapp
```

Выбери своего бота и укажи:

- **URL**: `https://dvinsk.lat`
- название, короткое описание, картинку 640×360
- короткое имя (short name), например `play` — из него получится ссылка вида
  `https://t.me/dvinsky_bridge_bot/play`

Эту ссылку и кидай в группу — по ней игра открывается внутри Telegram.

Дополнительно можно повесить кнопку «Играть» в меню бота:
`/setmenubutton` → выбрать бота → указать тот же URL.

## 3. Прописать токен на сервере

Без токена сервер не сможет проверить подпись Telegram и будет пускать
игроков **гостями** (без статистики). Игра при этом работает, но смысл теряется.

На VPS:

```bash
nano ~/.config/systemd/user/game125.service
```

Добавь строку в секцию `[Service]` (рядом с `Environment=ADMIN_LOGIN=...`):

```
Environment=TG_BOT_TOKEN=123456789:AAE...
```

Применить:

```bash
systemctl --user daemon-reload
systemctl --user restart game125
```

Проверить, что подхватилось:

```bash
systemctl --user show game125 -p Environment | grep -o TG_BOT_TOKEN | head -1
```

## 4. Проверить

Открой ссылку мини-приложения в Telegram. Должно быть так:

- игра открывается на весь экран, без ввода имени;
- в лобби рядом с именем стоит значок **аккаунт**;
- в личном кабинете копится статистика, ты попадаешь в топ-100.

Если просят ввести имя — токен не подхватился. Смотри логи:

```bash
journalctl --user -u game125 -n 30 --no-pager | grep tg
```

Строка `[tg] initData не прошла проверку — не задан TG_BOT_TOKEN` означает
ровно то, что написано.

## Как это устроено

- Telegram передаёт мини-приложению `initData` — данные профиля, **подписанные**
  токеном бота. Сервер проверяет подпись (HMAC-SHA256) и только после этого
  доверяет `telegram id`. Без проверки любой мог бы прислать чужой id и забрать
  чужой аккаунт, поэтому она обязательна.
- Подпись старше суток отклоняется (`TG_MAX_AGE_S`), чтобы перехваченную
  ссылку нельзя было использовать вечно.
- Аккаунт заводится сам при первом входе: `tg_id` в таблице `users`, логин вида
  `tg<id>`, пароль случайный — войти в такой аккаунт формой входа нельзя,
  только через Telegram.
- Обычные аккаунты (логин + пароль) работают как раньше, у них `tg_id` пустой.

## Ограничения

- Партия живёт в памяти сервера: при перезапуске (деплое) столы пропадают.
- Telegram кеширует мини-приложения: после обновления игры может понадобиться
  закрыть и открыть окно заново.
